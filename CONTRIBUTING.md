# Contributing

Thanks for helping make skill bundles portable and boringly safe.

## Development flow

1. Open an issue or small PR for focused changes.
2. Add or update fixtures for format behavior.
3. Run `npm run check`, `npm test`, `npm run smoke`, and `bash scripts/validate.sh`.
4. Keep PRs local-first: no telemetry, no surprise network calls.

## Code style

- TypeScript, strict mode.
- Prefer small pure functions with explicit errors.
- Keep CLI output scriptable and human-readable.

## Attribution

It is fine to acknowledge inspiration from public tools and workflows. Do not copy code, docs, fixtures, or branding without license review and explicit attribution.
